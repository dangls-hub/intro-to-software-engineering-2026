# Báo cáo dự án

## Mục đích

Thư mục này lưu các báo cáo phục vụ theo dõi tiến độ, tổng kết sprint và chuẩn bị nội dung nộp cuối kỳ cho dự án BlueMoon AMS.

## Các báo cáo nên có

| Tài liệu | Mục đích | Thời điểm cập nhật |
|---|---|---|
| Báo cáo tuần 1 | Tổng kết nền tảng, thiết kế, skeleton | 12/05/2026 |
| Báo cáo tuần 2 | Tổng kết chức năng chính và tích hợp API | 19/05/2026 |
| Báo cáo tuần 3 | Tổng kết kiểm thử, demo, bàn giao | 26/05/2026 |
| Báo cáo cuối kỳ | Trình bày toàn bộ quá trình và kết quả dự án | Khi nộp bài |

## Mẫu báo cáo tuần

```markdown
# Báo cáo tuần X

## Thời gian

- Từ ngày:
- Đến ngày:

## Mục tiêu tuần

-

## Công việc đã hoàn thành

| Thành viên | Công việc | Kết quả |
|---|---|---|
|  |  |  |

## Vấn đề gặp phải

| Vấn đề | Ảnh hưởng | Hướng xử lý |
|---|---|---|
|  |  |  |

## Kế hoạch tuần tiếp theo

-

## Đánh giá tiến độ

- Đúng tiến độ:
- Chậm tiến độ:
- Rủi ro:
```

## Dàn ý báo cáo cuối kỳ

1. Giới thiệu dự án.
2. Lý do chọn đề tài.
3. Mục tiêu và phạm vi.
4. Công nghệ sử dụng.
5. Phân công nhiệm vụ.
6. Phân tích yêu cầu.
7. Thiết kế hệ thống.
8. Kết quả triển khai.
9. Kiểm thử.
10. Hạn chế và hướng phát triển.
11. Kết luận.
